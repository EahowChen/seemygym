# 需求拆解（来自原始记录）

## 1. 产品目标

- 用表格方式记录不同训练动作的重量变化。
- 观察力量变化趋势，支持按动作进行可视化分析。

## 2. 关键信息架构

- 时间维度：Month -> Week -> Day。
- 训练维度：
  - 一级分类：训练部位（肩、胸、背、腿、腹...）
  - 二级分类：动作（侧举、平举...）
- 记录粒度：`某一天 x 某个动作 = 重量值(kg)`。

## 3. 功能需求

1. 导入年历
- 导入 2026 年（并可扩展至其他年份）
- 以 Day 为基础单位
- 通过纵向线条区分 Week 与 Month

2. Day 周几显示
- Day 旁可显示/隐藏周几
- 周几采用 7 种不同底色进行区分

3. 分类编辑
- 一级分类可编辑数量与名称
- 二级分类可编辑数量与名称
- 二级分类归属于对应一级分类

4. 训练记录
- 主表格按 Day 横向展开
- 每个动作有对应的每日重量输入

5. 结果可视化
- 每个动作行提供绘图入口
- 横轴：days，范围 0 ~ 最新记录天数
- 横轴需以虚线标出 week 与 month 的变化位置
- 横轴起点和终点显示日期标注，格式 `(D/M)`
- 纵轴：重量，范围 0 ~ (该动作最大值 + 10)
- 样式：Arial，无 grid，点线图

## 4. 第一版实现策略

- 前端单页原型（HTML + CSS + JavaScript）
- 本地内存状态管理，优先验证交互和数据结构
- 后续可扩展：
  - 本地持久化（LocalStorage/SQLite）
  - 导入导出（CSV/Excel）
  - 多用户与云同步
